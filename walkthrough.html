Por Dentro da Estratégia

Bem-vindo aos bastidores. Muitas ferramentas apenas "rodam". Eu projetei o RedRecon para pensar. Este walkthrough explica minha abordagem para transformar dados brutos em inteligência de risco, utilizando uma arquitetura modular e orientada a eventos.

O Desafio: Reduzir o ruído de milhares de logs.
Minha Solução: Um mecanismo de correlação que só alerta o que realmente importa para o negócio.

1. O Mecanismo de Parsing (A Inteligência)

O coração da ferramenta não é o scanner, é o Parser. Eu percebi que ferramentas como Nmap e Nuclei geram outputs muito diversos. Para unificá-los, criei um normalizador central.

FLUXO DE DADOS: DO CAOS À ESTRUTURA

graph LR A[Scanner Bruto] -->|Texto/XML| B(Engine Regex) B -->|Extração| C{Normalizador} C -->|Padronização| D[Objeto JSON Unificado] D -->|Análise| E[Banco de Dados de Risco] style A fill:#333,stroke:#666 style D fill:#2a4d2a,stroke:#4caf50 style E fill:#4d2a2a,stroke:#f44336

Isso me permite tratar um CVE crítico e uma porta aberta com a mesma estrutura de dados, facilitando a correlação posterior.

2. Análise de Risco e Matriz de Decisão

A automação cega é perigosa. Por isso, implementei uma Matriz de Risco Dinâmica. Não olho apenas para a severidade técnica (CVSS), mas para o contexto.

LÓGICA DE PRIORIZAÇÃO

graph TD Finding[Nova Descoberta] --> Context{É Exposta à Internet?} Context -- Sim --> HighRisk[Aumentar Criticidade] Context -- Não --> AvgRisk[Manter Score Base] HighRisk --> Chain{Faz parte de uma Chain?} Chain -- Sim --> CRITICAL[[ALERT] IMEDIATO] Chain -- Não --> REPORT[Relatório Diário] style CRITICAL fill:#770000,stroke:#ff0000,color:#fff

Exemplo Prático: Um servidor desatualizado na intranet é "Médio". O mesmo servidor exposto ao público torna-se "Alto". Se houver credenciais vazadas para ele, torna-se "Crítico".

3. Estudo de Caso: A "Cadeia Dante"

No Dashboard e no Relatório, você verá a detecção de um cenário real que simulei, apelidado de "Dante Chain". Isso demonstra a capacidade do RedRecon de conectar pontos aparentemente inofensivos:

  • 1. Info Leak (Baixo): Encontramos um log de aplicação acessível /var/log/app.log. Isoladamente, apenas uma falha de configuração.
  • 2. Serviço Desatualizado (Médio): Detectamos um Tomcat Manager antigo na porta 8080.
  • 3. Acesso Não Autorizado (Crítico): O RedRecon extraiu credenciais do log (Passo 1) e testou no Tomcat (Passo 2). Sucesso.

4. Como Construir o Seu Próprio

Se você deseja replicar essa arquitetura, comece simples:

  1. Escolha seu Wrapper: Python é ótimo para prototipar, Go para performance em escala.
  2. Defina seu Output: JSON deve ser o padrão universal.
  3. Falhe Graciosamente: Se o Nmap travar, seu script deve continuar. Use try/catch robustos e timeouts.
