Kerberoasting: The Silent Threat in Active Directory Environments

AD EXPLOITATION | 15 MIN READ | DEC 2025

Key Points

  • Basic Definition: A post-exploitation technique extracting Service Account hashes (SPNs) exploiting the Kerberos protocol.
  • Risks: Credential theft, lateral movement, and privilege escalation.
  • Ease of Execution: Any authenticated user/device can request TGS tickets without elevated privileges.
  • Mitigation: Passwords >25 chars, AES 256, monitoring Event ID 4769, and using gMSAs.

Kerberoasting represents one of the most subtle and persistent threats in Microsoft Active Directory (AD) environments. Identified as a form of credential theft (Tactic: Credential Access, Technique T1558 in MITRE ATT&CK), this technique allows attackers to obtain privileged credentials without generating immediate alerts. It is widely used by ransomware groups like Akira and APTs like Nobelium.

Historical and Technical Context

The Kerberos protocol, introduced in Windows 2000, operates via a Key Distribution Center (KDC). A design feature allows any authenticated user to request a Ticket-Granting Service (TGS) for services with registered SPNs. These tickets are encrypted with the service account's password hash. Attackers extract these tickets and crack the hashes offline (brute-force), away from the eyes of real-time monitoring.

Reports like the X-Force Threat Intelligence Index indicate a massive increase in these incidents, exploiting legacy environments and obsolete service accounts.

The Typical Attack Chain

The attack follows a logical and discreet sequence:

  1. Initial Compromise: Access to any domain user account (via phishing, etc.). No administrative privileges are required.
  2. Enumeration: Identifying accounts with non-null SPNs using BloodHound, PowerView, or LDAP queries.
    Get-NetUser -SPN | select samaccountname, serviceprincipalname
  3. Ticket Request (TGS): Using tools like Rubeus or GetUserSPNs.py (Impacket) to request tickets from the KDC.
  4. Extraction: Dumping tickets from memory (Kirbi or ccache format).
  5. Offline Cracking: Using Hashcat (mode 13100) or John the Ripper. Weak encryptions like RC4-HMAC (type 0x17) are cracked in hours.
Real World Impact: In the SolarWinds incident, attackers used Kerberoasting to escalate privileges after initial access, facilitating the creation of "Golden Tickets" for long-term persistence.

Prevention and Mitigation Strategies

Defending against Kerberoasting requires a layered approach ("Defense in Depth"):

  • Strong Passwords: Force service accounts to use passwords >25 characters. This makes brute-force computationally infeasible.
  • Adoption of AES: Disable RC4 encryption in the domain. Force AES-128/256 for Kerberos.
  • Managed Service Accounts (gMSA): Use Group Managed Service Accounts where possible. AD manages these passwords automatically (120 chars), eliminating manual risk.
  • Honeypots: Create fake service accounts with SPNs. Monitor Event ID 4769 for these accounts to detect recon attempts instantly.

Summary of Tools and Tactics

Phase Tool Objective
Recon PowerView, BloodHound Find targets with SPN set.
Execution Rubeus, Impacket Request and export TGS tickets.
Cracking Hashcat (Mode 13100) Recover plaintext password.
